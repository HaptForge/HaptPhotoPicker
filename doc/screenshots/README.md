# Screenshots

Drop your screenshot PNGs in this folder with the filenames below.
The root `README.md` references them by exact name — once the files
are saved (with the right names), they appear automatically on
GitHub and on pub.dev's package preview.

## Expected files

| Filename | What it should show |
|---|---|
| `gallery.png` | Main picker view: chrome + crop preview + tool launcher row + asset grid. The "hero" shot. |
| `crop.png` | Tool mode with the Crop tool open — bigger preview + aspect-ratio chips below. |
| `filter.png` | Tool mode with the Filter tool open — bigger preview + filter strip + intensity slider. |
| `rotate.png` | Tool mode with the Rotate tool open — rotate 90° / flip H / flip V pills + the Apple-style fine-rotation dial. |
| `adjust.png` | Tool mode with the Adjust tool open — manual brightness / contrast / saturation / exposure sliders. |

## Capture guidance

- **Device** — iPhone simulator at 1242×2688 (iPhone Pro Max) or
  similar. Real-device screenshots are fine too.
- **Theme** — light mode is the README default. If you also have
  dark-mode shots, save them as `*-dark.png` and we'll wire a
  light/dark toggle into the README later.
- **Privacy** — make sure no recognisable personal photos appear
  in the asset grid. The iOS simulator's seeded photo library is
  ideal because it ships with public-domain placeholder photos.
- **Crop the status bar** if you want a cleaner look, but full-frame
  is fine too — the README sets each image to 22% width so the
  device chrome stays small.

## Optional extras

If you want to add more screenshots beyond the four above (e.g.
`empty-state.png`, `multi-pick-badges.png`, `permission-denied.png`),
drop them in here and reference them from the README — they don't
need to be in the hero strip.
